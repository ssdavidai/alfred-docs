Alfred provides monitoring endpoints for health checks, system info, container status, log streaming, and Tailscale connectivity.

## Health Check

Get a health overview:

```bash
curl -X GET https://alfred-acme.tailnet.ts.net/api/v1/admin/health \
  -H "Authorization: Bearer YOUR_API_KEY"
```

**Response:**

The health endpoint runs `/opt/alfred/healthcheck.sh` on the instance. If the script returns valid JSON, it is passed through directly. Otherwise, the output is returned as `{ "raw": "<text>" }`.

```json
{ "raw": "STATUS: healthy\nContainers: 5/5 running\nDisk: 35% used\nMemory: 50% used" }
```

<Note>
The `alfred-init` container always shows `exited` with exit code 0. This is normal -- it runs once at startup to initialize the vault structure.
</Note>

## Health Status Interpretation

| Status | Meaning |
|--------|---------|
| **healthy** | All containers running, workers active, system resources OK |
| **degraded** | One or more workers stopped or containers unhealthy, but core services running |
| **down** | Critical containers stopped or system resources exhausted |

## System Information

Get system metrics (uptime, disk, memory):

```bash
curl -X GET https://alfred-acme.tailnet.ts.net/api/v1/admin/system/info \
  -H "Authorization: Bearer YOUR_API_KEY"
```

**Response:**

The system info endpoint returns raw text values from standard Linux commands:

```json
{
  "uptime": "up 1 day, 3 hours",
  "disk": "Filesystem      Size  Used Avail Use% Mounted on\n/dev/mapper/encrypted  100G   35G   65G  35% /mnt/encrypted",
  "memory": "              total        used        free      shared  buff/cache   available\nMem:           4096        2048        1024         128        1024        1792"
}
```

<Tip>
Parse the `disk` and `memory` fields to extract numeric values. Alert if disk usage exceeds 80%.
</Tip>

## Container Status

Check all container states:

```bash
curl -X GET https://alfred-acme.tailnet.ts.net/api/v1/admin/containers \
  -H "Authorization: Bearer YOUR_API_KEY"
```

**Response:**

Returns an array of container objects parsed from `docker compose ps --format json` (JSONL output). Fields use PascalCase to match Docker Compose's JSON output:

```json
[
  {
    "Name": "alfred-worker",
    "State": "running",
    "Status": "Up 24 hours",
    "ExitCode": 0
  },
  {
    "Name": "alfred-temporal",
    "State": "running",
    "Status": "Up 24 hours",
    "ExitCode": 0
  },
  {
    "Name": "alfred-init",
    "State": "exited",
    "Status": "Exited (0) 24 hours ago",
    "ExitCode": 0
  }
]
```

### Container Actions

Restart, stop, or start individual containers:

```bash
# Restart a container
curl -X POST https://alfred-acme.tailnet.ts.net/api/v1/admin/containers/alfred-worker/restart \
  -H "Authorization: Bearer YOUR_API_KEY"

# Stop a container
curl -X POST https://alfred-acme.tailnet.ts.net/api/v1/admin/containers/alfred-openclaw/stop \
  -H "Authorization: Bearer YOUR_API_KEY"

# Start a container
curl -X POST https://alfred-acme.tailnet.ts.net/api/v1/admin/containers/alfred-openclaw/start \
  -H "Authorization: Bearer YOUR_API_KEY"
```

**Response:**

```json
{
  "message": "Container \"alfred-worker\" restarted"
}
```

### Container Logs (Non-Streaming)

Get recent logs from a specific container:

```bash
curl -X GET "https://alfred-acme.tailnet.ts.net/api/v1/admin/containers/alfred-worker/logs?tail=100" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

**Response:**

```json
{
  "logs": "alfred-worker  | 2026-02-26 15:30:00 INFO curator: Processing inbox...\nalfred-worker  | 2026-02-26 15:30:15 INFO janitor: Scan complete\n..."
}
```

### Container States

| State | Description |
|-------|-------------|
| **running** | Container is active |
| **exited** | Container stopped (check `ExitCode`: 0 = normal, non-zero = error) |
| **restarting** | Container is restarting (possible crash loop) |
| **paused** | Container is paused |
| **dead** | Container is dead (requires manual intervention) |

## Log Streaming

Stream real-time logs using Server-Sent Events (SSE):

### Stream All Logs

```bash
curl -N https://alfred-acme.tailnet.ts.net/api/v1/logs \
  -H "Authorization: Bearer YOUR_API_KEY"
```

**Response (SSE stream):**

```
data: {"line":"alfred-worker  | 2026-02-26T15:30:00 INFO curator: Processing inbox file: meeting-notes.md"}

data: {"line":"alfred-worker  | 2026-02-26T15:30:15 INFO janitor: Scan complete: 3 issues found"}

data: {"line":"alfred-worker  | 2026-02-26T15:30:20 INFO distiller: Extracted 2 decisions from project/API Redesign.md"}
```

### Filter by Service

Stream logs from a specific service:

```bash
curl -N "https://alfred-acme.tailnet.ts.net/api/v1/logs?service=alfred" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

Available services correspond to Docker Compose service names:
- `alfred`
- `openclaw`
- `temporal`
- `alfred-aas`

### Tail Recent Logs

Get the last N lines before streaming:

```bash
curl -N "https://alfred-acme.tailnet.ts.net/api/v1/logs?tail=100" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

### Combined Filters

```bash
curl -N "https://alfred-acme.tailnet.ts.net/api/v1/logs?service=alfred&tail=50" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

<Tip>
SSE streams use `-N` (no buffer) with curl to receive events in real-time. For programmatic access, use an SSE client library. Each event contains a `line` field with the raw Docker Compose log line.
</Tip>

## Tailscale Status

Check Tailscale connectivity:

```bash
curl -X GET https://alfred-acme.tailnet.ts.net/api/v1/admin/tailscale/status \
  -H "Authorization: Bearer YOUR_API_KEY"
```

**Response:**

Returns the full JSON output from `tailscale status --json`:

```json
{
  "Self": {
    "DNSName": "alfred-acme.tail12345.ts.net.",
    "TailscaleIPs": ["100.64.1.23"],
    "Online": true
  },
  "Peer": { }
}
```

Check Tailscale Serve configuration:

```bash
curl -X GET https://alfred-acme.tailnet.ts.net/api/v1/admin/tailscale/serve \
  -H "Authorization: Bearer YOUR_API_KEY"
```

## Example Monitoring Workflows

### Health Check Script

Check health every 5 minutes and alert on degradation:

```bash
#!/bin/bash

API_KEY="YOUR_API_KEY"
BASE_URL="https://alfred-acme.tailnet.ts.net"
WEBHOOK="https://hooks.slack.com/services/YOUR/WEBHOOK/URL"

while true; do
  RESULT=$(curl -s -X GET "$BASE_URL/api/v1/admin/health" \
    -H "Authorization: Bearer $API_KEY")

  # The response may be { "raw": "..." } or structured JSON
  # Check for error indicators in the response
  if echo "$RESULT" | grep -qi "down\|error\|unhealthy"; then
    curl -X POST "$WEBHOOK" \
      -H "Content-Type: application/json" \
      -d "{\"text\":\"Alfred health check alert: $RESULT\"}"
  fi

  sleep 300
done
```

### Log Monitoring

Watch for errors in real-time:

```bash
curl -N "https://alfred-acme.tailnet.ts.net/api/v1/logs?tail=100" \
  -H "Authorization: Bearer YOUR_API_KEY" | \
  grep -i "error\|fatal\|exception"
```

### Resource Monitoring

Check disk and memory usage:

```bash
curl -s https://alfred-acme.tailnet.ts.net/api/v1/admin/system/info \
  -H "Authorization: Bearer YOUR_API_KEY" | jq .
```

The response has `uptime`, `disk`, and `memory` fields as raw text. Parse them as needed for alerting.

### Container Restart Detection

Monitor for container restarts by checking container states:

```bash
curl -s https://alfred-acme.tailnet.ts.net/api/v1/admin/containers \
  -H "Authorization: Bearer YOUR_API_KEY" | \
  jq '.[] | select(.State == "restarting") | .Name'
```

If a container is in a restart loop, check its logs:

```bash
curl -X GET "https://alfred-acme.tailnet.ts.net/api/v1/admin/containers/alfred-worker/logs?tail=200" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

## Error Handling

All errors follow a consistent format:

```json
{
  "error": {
    "code": "UNAUTHORIZED",
    "message": "Missing or malformed Authorization header"
  }
}
```

Error codes: `UNAUTHORIZED` (401), `VALIDATION_ERROR` (400), `NOT_FOUND` (404), `EXEC_ERROR` (500), `INTERNAL_ERROR` (500).

## Troubleshooting

### Health Endpoint Returns 500

The monitoring container itself may be down. SSH into the instance:

```bash
ssh deploy@alfred-acme.tailnet.ts.net
docker ps -a
```

Check if `alfred-aas` is running.

### Logs Not Streaming

Check if the AAS container is running:

```bash
curl -X GET https://alfred-acme.tailnet.ts.net/api/v1/admin/containers \
  -H "Authorization: Bearer YOUR_API_KEY"
```

If `alfred-aas` is down, restart it via SSH:

```bash
ssh deploy@alfred-acme.tailnet.ts.net
cd /opt/alfred/compose
docker compose restart alfred-aas
```

### Tailscale Disconnected

Check Tailscale status on the host:

```bash
ssh deploy@alfred-acme.tailnet.ts.net
sudo tailscale status
```

If disconnected, restart Tailscale:

```bash
sudo systemctl restart tailscaled
```

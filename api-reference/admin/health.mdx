---
title: "Run Health Check"
description: "Execute the healthcheck.sh script and return system health metrics"
api: "GET /api/v1/admin/health"
---

Run the health check script (`/opt/alfred/healthcheck.sh`). The response format depends on what the script outputs -- if the output is valid JSON it is returned parsed, otherwise it is returned as `{ "raw": "..." }`.

<RequestExample>

```bash curl
curl -H "Authorization: Bearer YOUR_API_KEY" \
  https://alfred-acme.tailnet.ts.net:3100/api/v1/admin/health
```

</RequestExample>

<ResponseExample>

```json
{
  "status": "ok",
  "containers": 5,
  "unhealthy": 0,
  "disk_percent": 45,
  "memory_percent": 62
}
```

</ResponseExample>

<Note>
  The response structure depends entirely on the `healthcheck.sh` script's output. The example above is representative but may differ on your instance. If the script outputs non-JSON text, the response will be `{ "raw": "..." }` instead.
</Note>
